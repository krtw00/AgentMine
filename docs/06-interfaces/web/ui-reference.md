# UI参考資料

タスク監視UIの設計参考。

## コンセプト

- **ワークフロー編集は不要** - AIがタスク分解を行う
- **監視・介入がメイン** - 実行状況の可視化、人間による介入ポイント
- **親子関係の表現** - タスク階層を視覚化

## 参考ツール

### 1. AgentOps
https://github.com/AgentOps-AI/agentops

AIエージェント監視専用プラットフォーム。

**良い点:**
- タイムライン/カスケードビュー
- セッションリプレイ
- LLM呼び出しとTool呼び出しの色分け
- コスト/トークン追跡
- エージェント一覧表示

**参考スクリーンショット:**
- Session Drill-Down: タイムラインでLLMイベントを可視化
- Session Replay: 詳細ビューでプロンプト内容も表示

### 2. Jaeger UI（分散トレーシング）
https://github.com/jaegertracing/jaeger-ui

**DevToolsネットワークパネル的なUI** - これが最も近いイメージ。

**良い点:**
- **ツリー構造**（折りたたみ可能）で親子関係を表現
- **ウォーターフォール**タイムラインで各スパンの実行時間を可視化
- クリックで**詳細展開**（Tags, Process, Logs）
- **エラー表示**（赤いマーク）
- **Depth**（階層の深さ）情報
- Service & Operation でラベリング

### 3. Dagster
https://github.com/dagster-io/dagster

**良い点:**
- DAGグラフでアセット依存関係を可視化
- シンプルで見やすいノード表示

### 4. Langfuse
https://github.com/langfuse/langfuse

**良い点:**
- LLM Observability専用
- トレースツリー表示
- 時間・コスト表示

## AgentMineへの適用

Jaeger UIのコンセプトをベースに設計：

| Jaeger | AgentMine |
|--------|-----------|
| Span | Task |
| Service & Operation | Agent & Task名 |
| Duration | 実行時間 |
| Tags/Logs | 入力/出力/ログ |
| エラーマーク | 失敗/要介入 |
| - | **＋介入ボタン**（一時停止、修正、承認、再実行） |

### UIの主要機能

1. **ツリービュー** - タスクの親子関係を折りたたみ可能なツリーで表示
2. **ウォーターフォール** - 各タスクの実行時間を横棒で可視化
3. **詳細パネル** - タスククリックで入力/出力/ログを表示
4. **介入ポイント** - 各タスクに介入ボタン
5. **ステータス表示** - 成功/失敗/進行中/要介入を色分け

### 介入機能

- **一時停止** - 実行中タスクを停止
- **再開** - 停止中タスクを再開
- **承認** - 人間による確認・承認
- **却下** - タスク結果を却下
- **修正** - タスクの入力/指示を編集して再実行
