# 目的・解決する課題

> Status: Active
> 最終更新: 2026-02-06

本ドキュメントは、プロジェクトの目的と解決する課題を明確にする。

---

## プロジェクトの目的

AgentMineの目的は、**AI Runnerに依存しない統合管理基盤**として、安全なスコープ制御・観測可能な事実に基づく状態管理・監査証跡を提供することである。

---

## 解決する課題

### 課題一覧

| # | 課題 | 影響 | 優先度 |
|---|------|------|--------|
| 1 | AIの暴走による意図しない変更 | 品質低下・セキュリティ事故 | High |
| 2 | 並列作業の状態不一致と衝突 | 進捗不明・衝突・再作業 | High |
| 3 | 特定AI Runnerへのベンダーロックイン | ツール切替不能・コスト最適化不可 | High |
| 4 | 判断根拠の不透明さ | 再現性低下・監査不能 | Medium |
| 5 | 実行状況の可視化不足 | 停止できない・介入が遅れる | Medium |
| 6 | 記憶の品質劣化 | 誤った学習の蓄積・長期運用の品質低下 | Medium |

### 課題の詳細

#### 課題1: AIの暴走による意図しない変更

| 項目 | 内容 |
|------|------|
| 現状 | AIが広範囲に書き換え、意図外の変更が混入する |
| 問題点 | 安全性が担保できず、人間のレビュー負担が増える |
| 解決策 | タスクごとのスコープ（write/exclude）とworktreeで物理的に制約する |

#### 課題2: 並列作業の状態不一致と衝突

| 項目 | 内容 |
|------|------|
| 現状 | 複数AIが同時実行すると状態が分散し追跡が困難。同一ファイルへの競合も発生する |
| 問題点 | タスクの進捗が不明確で、衝突が増える |
| 解決策 | DBマスターで状態を一元管理する。write_scopeの重なりを事前検出し、衝突を回避する |

#### 課題3: 特定AI Runnerへのベンダーロックイン

| 項目 | 内容 |
|------|------|
| 現状 | Claude Code、Codex等のAI開発ツールは各社専用である |
| 問題点 | タスク特性に応じたRunner選択ができない。コスト最適化も不可能 |
| 解決策 | RunnerAdapterで差異を吸収し、複数Runnerを統合管理する |

#### 課題4: 判断根拠の不透明さ

| 項目 | 内容 |
|------|------|
| 現状 | AIの報告や主観が混じり、判断が揺れる |
| 問題点 | 同条件でも結論が変わり、再現性が低い。監査時に根拠を示せない |
| 解決策 | 観測可能な事実を記録する。Run完了時にProof Bundle（証跡パック）を自動生成する |

#### 課題5: 実行状況の可視化不足

| 項目 | 内容 |
|------|------|
| 現状 | AIを端末で動かすと、実行状況が散らばる |
| 問題点 | 長時間実行の停止・再実行・介入が遅れる |
| 解決策 | Web UIでrunログと状態を統合し、stop/retry/continueで介入できるようにする |

#### 課題6: 記憶の品質劣化

| 項目 | 内容 |
|------|------|
| 現状 | AI実行から学んだ記憶が増え続け、古い/誤った情報が混在する |
| 問題点 | 誤った記憶が注入されることで、Workerの出力品質が劣化する |
| 解決策 | 記憶に信頼度・有効期限・承認を追加し、ガバナンスを適用する |

## 成功基準

| 基準 | 指標 | 目標値 |
|------|------|--------|
| 安全性 | スコープ外変更の未検出件数 | 0件 |
| 再現性 | 同条件での状態判定の一致率 | 100% |
| 可視性 | タスク/実行の追跡可能率 | 100% |
| AI非依存 | 2種類以上のRunner対応 | Claude + 1以上 |
| 衝突回避 | write_scope重複による並列衝突件数 | 0件 |

---

## 非目標

以下は本プロジェクトの目標ではない：

| 非目標 | 理由 |
|--------|------|
| AIが最終判断する仕組み | 判断は人間/AI上位層に委ねる |
| 単一AIの全自動開発 | 役割分離とレビュー前提を維持する |
| 特定AI Runnerの最適化 | AI非依存が原則。Runner固有の最適化はAdapterに閉じる |
| UI/UXの最適化 | MVPでは機能成立を優先する |
| Gitの置き換え | Git前提で共存する |
