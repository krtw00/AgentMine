# 目的・解決する課題

> Status: Draft
> 最終更新: 2026-02-01

本ドキュメントは、プロジェクトの目的と解決する課題を明確にする。

---

## プロジェクトの目的

AgentMineの目的は、
同じ条件なら同じようにAIが作業できるよう、ローカルで安全に実行し、
観測可能な事実をDBに集約して再現性のあるプロジェクト管理を提供することである。

---

## 解決する課題

### 課題一覧

| # | 課題 | 影響 | 優先度 |
|---|------|------|--------|
| 1 | AIの暴走による意図しない変更 | 品質低下・セキュリティ事故 | High |
| 2 | 並列作業の状態不一致 | 進捗不明・衝突・再作業 | High |
| 3 | 判断根拠の不透明さ | 再現性低下・監査不能 | Medium |
| 4 | 実行状況の可視化不足 | 停止できない・介入が遅れる | Medium |

### 課題の詳細

#### 課題1: AIの暴走による意図しない変更

| 項目 | 内容 |
|------|------|
| 現状 | AIが広範囲に書き換え、意図外の変更が混入する |
| 問題点 | 安全性が担保できず、人間のレビュー負担が増える |
| 解決策 | タスクごとのスコープ（write/exclude）とworktreeで物理的に制約する |

#### 課題2: 並列作業の状態不一致

| 項目 | 内容 |
|------|------|
| 現状 | 複数AIが同時実行すると状態が分散し追跡が困難 |
| 問題点 | タスクの進捗が不明確で、衝突が増える |
| 解決策 | DBマスターで状態を一元管理し共有する |

#### 課題3: 判断根拠の不透明さ

| 項目 | 内容 |
|------|------|
| 現状 | AIの報告や主観が混じり、判断が揺れる |
| 問題点 | 同条件でも結論が変わり、再現性が低い |
| 解決策 | 観測可能な事実を記録し、判断材料を固定する |

#### 課題4: 実行状況の可視化不足

| 項目 | 内容 |
|------|------|
| 現状 | AIを端末で動かすと、実行状況が散らばる |
| 問題点 | 長時間実行の停止・再実行・介入が遅れる |
| 解決策 | Web UIでrunログと状態を統合し、stop/retry/continueで介入できるようにする |

---

## 成功基準

| 基準 | 指標 | 目標値 |
|------|------|--------|
| 安全性 | スコープ外変更の未検出件数 | 0件 |
| 再現性 | 同条件での状態判定の一致率 | 100% |
| 可視性 | タスク/実行の追跡可能率 | 100% |

---

## 非目標

以下は本プロジェクトの目標ではない：

| 非目標 | 理由 |
|--------|------|
| AIが最終判断する仕組み | 判断は人間/AI上位層に委ねる |
| 単一AIの全自動開発 | 役割分離とレビュー前提を維持する |
| UI/UXの最適化 | MVPでは機能成立を優先する |
| Gitの置き換え | Git前提で共存する |
